# 台南公車通知 Bot — 部署與環境變數設定指南

> 這份文件是「照著做」的操作手冊。你只需要到指定平台的**網頁介面**點選、建立、複製資料，
> 再貼到指定位置即可，**不需要在自己的電腦打任何終端機指令**。

---

## 0. 核心觀念：機密只放兩個地方

| 位置 | 用途 | 存放什麼 |
|------|------|----------|
| 專案根目錄的 `.env` 檔 | 本地開發／我幫你在本機測試與執行 | 全部 4 個機密（TDX×2、Telegram×1、以及 2 組隨機密鑰） |
| GCP **Secret Manager** | 雲端正式環境（Cloud Run）執行 | 同樣那 5 個機密（雲端不讀 `.env`） |

你要親自「取得」的原始資料其實只有 **3 樣**（TDX 金鑰、Telegram token），另外 2 組隨機密鑰我已幫你產生好、直接沿用即可。

### 需要的環境變數總表

| 變數名稱 | 是什麼 | 去哪取得 | 放 `.env` | 放 Secret Manager |
|----------|--------|----------|:--------:|:-----------------:|
| `TDX_CLIENT_ID` | TDX 公車 API 金鑰 ID | TDX 會員中心 | ✅ | ✅ |
| `TDX_CLIENT_SECRET` | TDX 公車 API 金鑰密碼 | TDX 會員中心 | ✅ | ✅ |
| `TELEGRAM_BOT_TOKEN` | Telegram bot token | @BotFather | ✅ | ✅ |
| `TICK_AUTH_TOKEN` | 保護 `/tick` 的隨機密鑰 | 已幫你產生 | ✅ | ✅ |
| `TELEGRAM_WEBHOOK_SECRET` | 保護 `/webhook` 的隨機密鑰 | 已幫你產生 | ✅ | ✅ |

## 1. 平台 A：TDX（取得公車到站 API 金鑰）

**為什麼需要**：本專案每次查「70左／70右 幾分鐘到站」都是呼叫 TDX 的「公車預估到站」API
（`公共運輸-公車 → 基礎服務`）。免費會員的基礎服務額度就足夠，本專案每月約 858 次呼叫，遠低於額度。

**這個專案只需要這一組金鑰，你把 Client Id / Client Secret 拿回來給我就可以了。**

操作步驟（全在網頁）：
1. 前往 <https://tdx.transportdata.tw/>，右上角「註冊 / 登入」→ 註冊會員。
2. 到信箱收驗證信、完成 Email 驗證，等待帳號審核通過後登入。
3. 登入後點右上角帳號 →「會員中心」→ 左側「API 金鑰管理」。
4. 按「建立金鑰」，命名（例如 `tainan-bus`）→ 建立。
5. 複製這組的 **Client Id** 與 **Client Secret**（每位會員最多可建 3 組）。
6. 貼到 `.env` 的 `TDX_CLIENT_ID`、`TDX_CLIENT_SECRET`。

> 補充：認證方式為 OIDC Client Credentials，token 效期 1 天，程式會自動換發並快取，你不用管。

## 2. 平台 B：Telegram（建立 bot）

操作步驟（全在 Telegram App）：
1. 在 Telegram 搜尋並打開 [@BotFather](https://t.me/BotFather)。
2. 送 `/newbot`，依指示輸入 bot 顯示名稱與 username（username 需以 `bot` 結尾）。
3. BotFather 會回一串 **token**（形如 `123456789:AAxxxxxxxx...`）。
4. 把 token 貼到 `.env` 的 `TELEGRAM_BOT_TOKEN`。
5. **重要**：用你自己的 Telegram 帳號打開這個新 bot，按一次 `/start`（之後才收得到推播）。
   親友要用，也各自 `/start` 一次即可。

## 5. 平台 C：GCP（雲端正式環境，全部在瀏覽器 Console 操作）

前往 <https://console.cloud.google.com/>，用 Google 帳號登入。以下每一步都是網頁點選。

### 5.1 建立專案
- 頂端專案選單 →「新增專案」→ 命名（例如 `tainan-bus`）→ 建立。
- 記下 **Project ID**（形如 `tainan-bus-470xxx`）。

### 5.2 啟用需要的 API
- 左側選單搜尋「API 和服務 → 已啟用的 API 和服務 → 啟用 API 和服務」，
  逐一搜尋並「啟用」以下六項：
  - Cloud Run Admin API
  - Cloud Build API
  - Firestore API
  - Secret Manager API
  - Artifact Registry API
  - Cloud Scheduler API

### 5.3 建立 Firestore 資料庫（存使用者設定與當日狀態）
- 搜尋「Firestore」→ 建立資料庫 → 選 **原生模式（Native）** → 位置選 `asia-east1`（台灣）→ 建立。

### 5.4 建立 5 個 Secret（Secret Manager）
- 搜尋「Secret Manager」→「建立密鑰」，重複 5 次，**名稱要和 `.env` 完全一致**：
  `TDX_CLIENT_ID`、`TDX_CLIENT_SECRET`、`TELEGRAM_BOT_TOKEN`、`TICK_AUTH_TOKEN`、`TELEGRAM_WEBHOOK_SECRET`。
- 每個的「密鑰值」貼上與 `.env` 相同的值 → 建立。

### 5.5 建立 Cloud Run 服務（同時設定自動部署 CI/CD）
- 搜尋「Cloud Run」→「部署容器 → 服務」。
- 選 **「從原始碼儲存庫持續部署」**（Continuously deploy from a repository）→「設定 Cloud Build」。
  - 連結你的 GitHub 帳號 → 選這個專案的 repo → 分支填 `^main$`。
  - 建置類型選 **Dockerfile**（專案根目錄已有 `Dockerfile`）→ 儲存。
- 服務設定：
  - 區域：`asia-east1`
  - 驗證：勾選 **「允許未經驗證的叫用」**（`/webhook` 需讓 Telegram 從外部打進來；安全性由下方密鑰標頭把關）。
- 展開「容器、磁碟區、網路、安全性」→「變數與密鑰」分頁：
  - 用「參照密鑰」把上面 5 個 Secret 逐一對應到**同名**的環境變數。
- 按「建立」。第一次會自動建置並部署；之後你 **push 到 `main` 就會自動重新部署**（這就是 CI/CD，不需在 GitHub 存任何金鑰）。

### 5.6 給 Cloud Run 服務帳號授權
- 到「IAM 與管理 → IAM」，找到 Cloud Run 使用的服務帳號（預設是 Compute Engine 預設服務帳號 `...-compute@developer.gserviceaccount.com`）。
- 點編輯 → 新增兩個角色：
  - **Secret Manager 密鑰存取者**（Secret Manager Secret Accessor）
  - **Cloud Datastore 使用者**（Datastore User，Firestore 用）
- 儲存。

### 5.7 取得服務網址
- 回 Cloud Run 服務頁，複製最上方的網址（形如 `https://tainan-bus-xxxx.a.run.app`）。
- 用瀏覽器開 `<網址>/health`，看到 `{"ok":true}` 代表服務活著。

## 6. 平台 C 續：Cloud Scheduler（每 5 分鐘觸發推播檢查）

- 在 GCP Console 搜尋「Cloud Scheduler」→「建立工作」。
  - 名稱：`tainan-bus-tick`
  - 區域：`asia-east1`
  - 頻率：`*/5 * * * *`
  - 時區：`台北 (Asia/Taipei)`
  - 目標類型：**HTTP**
  - 網址：`<你的 Cloud Run 網址>/tick`
  - HTTP 方法：`POST`
  - 展開「其他設定 → 新增標頭」：
    - 名稱 `X-Tick-Token`，值填 `TICK_AUTH_TOKEN` 的內容（即 `.env` 裡那串）。
  - 建立。

> 觸發時若不在推播時段（例如半夜）不會推播，屬正常。

---

## 7. 綁定 Telegram webhook（讓你的訊息打到服務）

只需做一次，**用瀏覽器貼網址即可，不用終端機**：

在瀏覽器網址列貼上（把 `<TOKEN>`、`<網址>`、`<SECRET>` 換成實際值）後按 Enter：

```
https://api.telegram.org/bot<TOKEN>/setWebhook?url=<你的Cloud Run網址>/webhook&secret_token=<TELEGRAM_WEBHOOK_SECRET>
```

看到回應 `{"ok":true,"result":true,...}` 即成功。
（`<TOKEN>` = `TELEGRAM_BOT_TOKEN`；`<SECRET>` = `TELEGRAM_WEBHOOK_SECRET`。若不想自己貼，把 `.env` 填好後我幫你送一次。）

## 8. 端到端驗證

1. 瀏覽器開 `<網址>/health` → `{"ok":true}`。
2. Telegram 對你的 bot 送 `/start` → 應收到歡迎詞與底部常駐設定鍵盤。
3. 隨意按底部「⏱ 推播間隔 / 🚏 推播公車站 / 📅 推播時間」確認有反應。
